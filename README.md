# Pet Project: Data Warehouse для ресторана (OLTP → Star Schema DWH)

**Цель проекта**  
Разработать оперативную базу данных ресторана (OLTP) по полному ТЗ и построить на её основе аналитическое хранилище данных (DWH) с использованием dimensional modeling (Kimball — Star Schema).

### Что было сделано

**1. OLTP-система (оперативная база)**
- Спроектирована и нормализована база данных ресторана (3НФ)
- Реализована иерархия категорий блюд, история смены должностей сотрудников, система скидок клиентов
- Создано 15+ таблиц с правильными ограничениями и внешними ключами
- Написано 50+ SQL-запросов различной сложности

**2. DWH (Data Warehouse) — Star Schema**
- Построена dimensional модель по методологии Kimball
- Реализован процесс ETL из OLTP в DWH
- Подготовлена структура для аналитических отчётов

**3. Аналитические запросы на основе хранилища**
- Выручка по месяцам (сравнение с предыдущим годом)
- ТОП-10 самых прибыльных блюд за всё время
- Эффективность официантов (выручка + количество заказов)
- Анализ скидок — сколько теряем на скидках
- Любимые блюда клиентов
- Загрузка залов по дням недели
- Динамика отменённых заказов

**Технологии**

- Erwin Data Modeler (ER-диаграммы)
- Dimensional Modeling (Star Schema)
